# German Editorial Guidelines

Updated: 2026-06-26
Scope: unpublished German source pages for `ilovemontefiascone.com`.

## Voice

- Use natural German for culturally curious travellers, not literal translations.
- Prefer calm, informative travel-guide language over promotional claims.
- Keep Montefiascone as the central base for Lake Bolsena, Tuscia and northern Lazio.
- Explain Italian place names clearly, but keep proper names such as Rocca dei Papi, Basilica di San Flaviano, Est! Est!! Est!!! and Via Francigena.
- Avoid keyword stuffing. Use entities because they help orientation, not because they repeat search terms.

## Accuracy

- Do not invent opening hours, prices, booking rules, transport timetables, event programmes or exact access conditions.
- Use cautious phrases for variable data: "vor der Reise prüfen", "je nach Saison", "direkt beim Anbieter prüfen".
- Do not present this website as official. Keep the independent editorial disclaimer.

## Structure

- Preserve existing layout, CSS classes, header, footer, breadcrumbs, FAQ and JSON-LD patterns.
- Use one H1 per page, followed by logical H2 sections.
- Add concise "Kurz gesagt" bullets on practical pages.
- Keep CTAs as internal planning links, not commercial pressure.

## Technical Rules

- DE source pages use `lang="de"`.
- Slugs use ASCII transliteration: `ue`, `ae`, `oe`, `ss`; visible text uses correct German characters.
- Do not add public DE hreflang until DE is published.
- Do not add `/de/` to sitemap, `llms.txt` or the public build.
- Check every page for mojibake before future publication.
