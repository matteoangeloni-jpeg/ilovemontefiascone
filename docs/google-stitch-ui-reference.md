# Google Stitch UI Reference

> Riferimento visuale esterno usato solo come moodboard. Nessun asset, immagine o testo di Google Stitch va pubblicato nel sito.

## Sorgenti consultate

- Cartelle locali esterne al repository con naming `stitch_montefiascone_tourism_portal*`.
- Riferimenti letti:
  - `lake_stone/DESIGN.md`
  - `montefiascone_homepage_updated/screen.png`

## Cosa prendere come riferimento

| Categoria | Stato | Nota |
| --- | --- | --- |
| Layout reference | `yes` | Hero ampio, card editoriali, blocchi con gerarchia chiara. |
| Palette reference | `yes` | Borgogna, tufo, blu lago, antracite gia coerenti con il design system del sito. |
| Spacing reference | `yes` | Più respiro tra hero, card, sidebar e footer. |
| Typography reference | `partial` | Manteniamo `Playfair Display` + `Inter`, senza introdurre font esterni diversi. |
| Header/footer reference | `yes` | Sticky header leggero, footer piu ordinato e sezionato. |
| Menu/dropdown reference | `yes` | Dropdown per cluster con label brevi e pannelli leggibili. |
| Card/hero reference | `yes` | Contrasto forte tra foto e overlay informativi, senza sovraccaricare il contenuto. |
| Do-not-use asset | `mandatory` | Tutte le immagini Stitch, i testi placeholder e qualsiasi export generato restano esclusi dal pubblico. |

## Elementi effettivamente tradotti nel refresh

- Header più trasparente e consistente tra lingue.
- Dropdown clusterizzati con pannelli compatti e leggibili su mobile.
- Footer con struttura stabile: brand, sezioni link, nota editoriale.
- Hero più fotografici sui cluster chiave, ma solo con `.webp` reali provenienti dallo stash manuale.

## Elementi esplicitamente esclusi

- Immagini export o mockup Google Stitch.
- Testi generati o placeholder Stitch.
- Pattern troppo “template-like” non coerenti con il sito reale.
- Qualsiasi asset remoto, base64 o AI non verificato.
