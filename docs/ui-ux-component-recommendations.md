# UI / UX Component Recommendations

## Hero

### Stato attuale

- La hero funziona bene quando combina:
  - H1 forte;
  - eyebrow contestuale;
  - immagine riconoscibile;
  - summary box laterale.

### Problema

- In alcuni eventi singoli l’immagine è troppo generica.
- Nella homepage la hero è forte, ma viene seguita da molti blocchi con pari importanza.

### Raccomandazione

- Mantenere il pattern attuale.
- Distinguere meglio le hero “hub” da quelle “spoke”.
- Nelle pagine evento usare immagini più descrittive del singolo appuntamento quando disponibili.

### Priorità

`P2`

## Card

### Stato attuale

- Le card sono il componente più utile del portale.
- Funzionano bene per:
  - spoke monumento;
  - mini-guide;
  - FAQ laterali;
  - blocchi “in breve”.

### Problema

- Nei cluster principali alcune card sembrano tutte di uguale peso.
- In DE il testo lungo rischia wrap visivo più pesante.

### Raccomandazione

- Introdurre una gerarchia visiva più netta tra:
  - card primaria;
  - card di servizio;
  - card correlata.

### Priorità

`P1`

## Blocchi correlati

### Stato attuale

- Presenti e SEO-friendly.

### Problema

- Talvolta sono troppo generici.
- In alcuni hub danno l’impressione di “altre liste” più che di percorso successivo.

### Raccomandazione

- Ogni blocco correlato deve rispondere a una sola domanda:
  - cosa apro dopo?
  - cosa apro se resto una notte?
  - cosa apro se cerco info pratiche?

### Priorità

`P1`

## Event card

### Stato attuale

- Buona presenza di date, luogo o focus evento.

### Problema

- La differenza tra hub eventi, calendario e scheda evento non è sempre immediata.

### Raccomandazione

- Rendere più visibili:
  - data;
  - stato informativo;
  - relazione con l’hub.

### Priorità

`P1`

## Monument / place page

### Stato attuale

- È la tipologia più riuscita del portale.

### Problema

- In pochi casi i percorsi “continua con” tornano a hub troppo ampi.

### Raccomandazione

- Conservare il layout.
- Migliorare solo la precisione dei link laterali e delle uscite.

### Priorità

`P2`

## Wine / gastronomy page

### Stato attuale

- Il cluster è ricco e credibile.

### Problema

- Il layout non distingue sempre bene:
  - hub introduttivo;
  - guida pratica;
  - spoke esperienziale;
  - evento collegato.

### Raccomandazione

- Usare CTA più esplicite:
  - “apri la guida cantine”;
  - “vai alla Fiera del Vino”;
  - “scopri la cucina tipica”.

### Priorità

`P1`

## Mappa

### Stato attuale

- Pagina utile e semantica.

### Problema

- Le uscite verso le altre guide non sono sempre il passo successivo più chiaro.

### Raccomandazione

- Trattare la mappa come centro di smistamento pratico:
  - arrivo;
  - salita/dislivello;
  - lago;
  - Francigena;
  - parcheggi;
  - panorami.

### Priorità

`P1`

## CTA

### Stato attuale

- Presenti in quasi tutto il corpus.

### Problema

- Alcune CTA sono semanticamente generiche.

### Raccomandazione

- Standardizzare 3 famiglie:
  - CTA hub;
  - CTA spoke;
  - CTA pratica.

### Priorità

`P1`

## Footer

### Stato attuale

- Trasmette bene indipendenza editoriale.

### Problema

- Alcuni footer sono molto minimali, altri più ricchi.

### Raccomandazione

- Allineare la profondità minima del footer nei template secondari.

### Priorità

`P2`

## Language switcher

### Stato attuale

- Generalmente chiaro e ben posizionato.

### Problema

- Assente nei 3 blocker DE.

### Raccomandazione

- Considerarlo parte obbligatoria del frame globale di pagina.

### Priorità

`P0`

## Mobile navigation

### Stato attuale

- L’impostazione hamburger è coerente.

### Problema

- Il valore della navigazione cambia troppo tra template completi e template ridotti.

### Raccomandazione

- Stabilire un livello minimo comune di voci in tutte le lingue.

### Priorità

`P1`

## Breadcrumb

### Stato attuale

- Molto utile dove presente.

### Problema

- Mancante o non uniforme in una parte del corpus EN/DE.

### Raccomandazione

- Renderlo obbligatorio in spoke, guide, day trip, practical pages ed eventi singoli.

### Priorità

`P1`

## FAQ

### Stato attuale

- Buon equilibrio tra leggibilità e SEO.

### Problema

- Non è un problema di componente, ma di distribuzione: alcune pagine forti ne beneficiano, altre meno.

### Raccomandazione

- Tenerle dove aiutano davvero l’utente a chiudere il dubbio operativo.

### Priorità

`P3`

## Immagini

### Stato attuale

- Immagini spesso corrette e ben descritte.

### Problema

- In alcuni spoke evento l’immagine è più “di contesto” che “di evento”.
- In qualche pagina community/sport l’immagine informativa è corretta ma povera come hero.

### Raccomandazione

- Non aumentare il numero immagini.
- Migliorare invece il match tra immagine, promesse del titolo e micro-intento della pagina.

### Priorità

`P2`
